# Contributing Guidelines

To ensure a smooth and effective collaboration, please follow these guidelines when contributing to psychoroid.com:

1. **Code Style**: Follow the existing code style and conventions used in the psychoroid.com codebase. We use [Prettier](https://prettier.io/) for code formatting and [ESLint](https://eslint.org/) for linting.

2. **Documentation**: Update the relevant documentation when making changes to the codebase. This includes inline comments, README files, and API documentation.

3. **Testing**: Write unit tests for new features and bug fixes. Ensure that all tests pass before submitting a pull request.

4. **Commit Messages**: Write clear and descriptive commit messages that explain the purpose of your changes. Use the present tense and imperative mood (e.g., "Add feature" instead of "Added feature").

5. **Pull Requests**: Keep pull requests focused on a single feature, bug fix, or improvement. Provide a detailed description of your changes and reference any relevant issues.

6. **Code Reviews**: Be open to feedback and constructive criticism during the code review process. Address any comments or suggestions made by the maintainers.

7. **Licensing**: By contributing to developers.psychoroid.com, you agree that your contributions will be licensed under the [MIT License](https://github.com/psychoroid/developers.psychoroid/blob/main/LICENSE).

We appreciate your adherence to these guidelines and your efforts to make developers.psychoroid.com a better platform for everyone.
