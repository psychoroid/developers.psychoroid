# lomi. CLI

The lomi. CLI allows you to interact with your lomi. account and perform various tasks related to your payment integration setup using simple command-line commands.

This makes it easier to manage your payment integrations, test your setup in different environments, automate your testing process, and deploy the applicaiton more easily.

## Installation

To install the lomi. CLI, use the package manager of your choice, such as NPM:

```bash copy
npm install -g @lomi/cli
```

## Authentication

To use the lomi. CLI, you need to authenticate with your lomi. account. You can do this by setting the `LOMI_API_KEY` environment variable to your API key:

```bash copy
export LOMI_API_KEY=your_api_key_here
```

Alternatively, you can pass the API key as a command-line argument using the `--api-key` flag:

```bash copy
lomi --api-key your_api_key_here command
```

## Commands

### `lomi setup`

The `setup` command helps you set up your lomi. integration by guiding you through the process of creating a merchant account, setting up payment methods, and configuring your integration settings.

```bash copy
lomi setup
```

### `lomi test`

The `test` command allows you to test your payment integration by simulating transactions and verifying that your integration is working as expected.

```bash copy
lomi test
```

### `lomi transactions`

The `transactions` command allows you to view and manage your transactions. You can list recent transactions, view transaction details, and perform actions such as refunds or captures.

```bash copy
lomi transactions list
lomi transactions view <transaction_id>
lomi transactions refund <transaction_id>
lomi transactions capture <transaction_id>
```

### `lomi webhooks`

The `webhooks` command allows you to manage your webhooks. You can list your registered webhooks, create new webhooks, and delete existing webhooks.

```bash copy
lomi webhooks list
lomi webhooks create --url <webhook_url> --events <comma_separated_events>
lomi webhooks delete <webhook_id>
```
